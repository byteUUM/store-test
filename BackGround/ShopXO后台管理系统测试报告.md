## 4. 主要缺陷

### BUG-001 商品搜索存在反射型 XSS

| 字段 | 内容 |
| --- | --- |
| 严重级别 | 高 |
| 模块 | 商品管理 |
| URL | `adminkpeyv4.php?s=goods/index.html&f2p=<script>alert(1)</script>` |
| 前置条件 | 后台管理员已登录 |
| 实际结果 | 页面将 `f2p` 参数中的 `<script>alert(1)</script>` 反射到 DOM 中，并触发 `alert(1)` |
| 期望结果 | 输入内容应被转义或过滤，不应作为可执行脚本进入页面 |
| 影响 | 攻击者可构造链接诱导管理员访问，执行恶意脚本，可能窃取会话、执行后台操作或篡改页面内容 |
| 建议 | 对所有查询参数做 HTML 转义；后端输出统一使用安全模板转义；增加 CSP；对搜索输入做白名单或安全过滤 |

复核证据：

- 自动化检测状态：`FAIL_XSS_REFLECTED`
- 弹窗记录：`alert`，消息为 `1`
- DOM 片段中出现：`<script>alert(1)</script>`

## 5. 观察项

### OBS-001 未登录访问使用脚本跳转，HTTP 状态仍为 200

未登录访问后台首页、商品页、用户页、SQL控制台、删除接口时，响应内容为跳转脚本：

```js
window.location.href="http://49.235.61.184/adminkpeyv4.php?s=admin/logininfo.html";
```

功能上能拦截未登录访问，但 HTTP 状态码仍为 200。建议后端直接返回 302 跳转登录页，或接口返回 401/403，便于安全设备、监控和自动化测试准确识别鉴权失败。

### OBS-002 部分页面预期关键字与实际页面文案不一致

以下页面加载成功，但自动化关键字检查出现警告，人工判断不属于页面不可用：

| 页面 | 观察 |
| --- | --- |
| 协议管理 | 页面包含协议编辑内容和保存入口，但未出现“新增”按钮 |
| 商品评论 | 当前无评论数据，表头为用户信息/评论内容/评分等 |
| 订单售后 | 页面显示售后相关字段，但未直接出现“售后”字样 |
| 地区管理 | 页面显示省市地区树数据，但自动关键字未命中 |
| 品牌分类 | 页面存在编辑/删除入口，未出现“新增”按钮 |
| 文章分类 | 页面存在编辑/删除入口，未出现“新增”按钮 |

建议后续确认这些页面是否本来就不支持新增，还是当前账号/数据状态导致入口隐藏。

### OBS-003 高风险入口存在，需要重点回归

本轮只检查入口，不执行操作。后续正式回归建议重点覆盖：

| 模块 | 高风险入口 |

## 9. 测试结论

本轮后台非破坏性测试通过了大部分页面可用性、菜单可达性和基础鉴权检查，系统整体可用。当前最需要优先修复的是商品管理搜索参数的反射型 XSS，该问题已确认可执行脚本，建议作为高优先级安全缺陷处理。

修复后建议进行二次回归：

1. 回归 `goods/index.html` 所有筛选字段的输出转义。
2. 扩展检查用户、订单、文章、品牌、仓库等列表页的搜索参数是否存在同类 XSS。
3. 对删除、状态更新、缓存清理、应用导入、SQL控制台做权限和 CSRF 专项测试。
4. 补充真实新增/编辑/删除流程测试，使用测试数据并提前约定回滚策略。
